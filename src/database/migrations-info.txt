Tutorial: Flujo Básico de Migraciones en TypeORM (Usando Comandos Personalizados)
-------------------------------------------------------------------------------

Este tutorial explica cómo generar, ejecutar y revertir migraciones en TypeORM usando comandos personalizados para diferentes bases de datos.


-- Paso 1: Generar el archivo de migración -----------------------------------------------------------------

Para crear una migración, usa el siguiente comando, donde "main" corresponde a las migraciones de cierta base de datos.
(VER PACKAGE JSON PARA VER CUALES HAY DISPONIBLES)

Ejemplo para la base de datos "main":

npm run migration:generate:main -- --name="CreateTable"

Esto generará un archivo de migración en "src/database/migrations/main/" con un nombre similar a "1678886400000-CreateTable.ts".

OJO: La ruta de creacion de la migracion tambien esta en el comando del package json respectivo


-- Paso 2: Ejecutar la migración -----------------------------------------------------------------

Para aplicar los cambios a la base de datos que se desea, se debe ejecutar:

npm run migration:run:main

Esto ejecutará el método up() de las migraciones pendientes en la base de datos "main".

-- Paso 4 (Opcional): Revertir la migración --

Si necesitas deshacer el último cambio, ejecuta:

npm run migration:revert:main

Esto ejecutará el método down() de la última migración aplicada en la base de datos "main".

Recuerda: Puedes cambiar "main" por otro parametro que se encuentre en los scripts del package json
